## Rules to Follow:

### 1. Model
- Use exact field names, types, and defaults as defined in schema
- Add indexes on: organization_id, uid, created_at
- Use { timestamps: true } where applicable

### 2. Routes (for each collection)
- GET    /[collection]/getAll
- GET    /[collection]/getById/:id
- GET    /[collection]/getByOrg/:organization_id
- POST   /[collection]/create
- POST   /[collection]/update
- POST   /[collection]/delete

### 3. Controller (for each collection)
- Use async/await with try/catch
- Always return this response format:
  {
    "status": true | false,
    "message": "description",
    "data": {} | [] | null
  }
- Add pagination: ?page=1&limit=10
- Always filter by organization_id for multi-tenant security
- Validate required fields before saving

### 4. Auth Middleware
- Verify JWT from header: x-access-token
- Attach user data to req.user
- Protect all routes except public ones

### 5. Error Handling
- 400 → Validation error
- 401 → Unauthorized / token missing or invalid
- 404 → Record not found
- 500 → Internal server error
